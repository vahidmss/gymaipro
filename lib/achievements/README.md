# سیستم دستاوردها (Achievements System)

## 📋 معرفی

سیستم دستاوردها یک سیستم جامع و کامل برای مدیریت دستاوردهای کاربران است که شامل:
- ذخیره‌سازی پیشرفت دستاوردها در دیتابیس Supabase
- Cache محلی با SharedPreferences برای عملکرد آفلاین
- سینک خودکار بین cache محلی و دیتابیس
- مدیریت امتیازات و تاریخچه

## 🗄️ ساختار دیتابیس

### جدول `achievements`

```sql
CREATE TABLE achievements (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES profiles(id),
  achievement_id VARCHAR(100) NOT NULL,
  current_value INTEGER NOT NULL DEFAULT 0,
  unlocked_at TIMESTAMP WITH TIME ZONE,
  created_at TIMESTAMP WITH TIME ZONE,
  updated_at TIMESTAMP WITH TIME ZONE,
  UNIQUE (user_id, achievement_id)
);
```

### Indexes

- `achievements_user_id_idx` - برای جستجوی سریع دستاوردهای یک کاربر
- `achievements_achievement_id_idx` - برای جستجوی سریع یک دستاورد خاص
- `achievements_unlocked_at_idx` - برای فیلتر دستاوردهای باز شده

### RLS Policies

- کاربران فقط می‌توانند دستاوردهای خود را مشاهده کنند
- کاربران فقط می‌توانند دستاوردهای خود را ایجاد/به‌روزرسانی کنند
- کاربران فقط می‌توانند دستاوردهای خود را حذف کنند

## 🔧 نحوه استفاده

### 1. راه‌اندازی دیتابیس

ابتدا باید جدول را در Supabase ایجاد کنید:

```bash
# اجرای فایل SQL
psql -h your-supabase-host -U postgres -d postgres -f sql/create_achievements_table.sql
```

یا از طریق Supabase Dashboard:
1. به بخش SQL Editor بروید
2. محتوای `sql/create_achievements_table.sql` را کپی کنید
3. اجرا کنید

### 2. استفاده در کد

```dart
// دریافت سرویس
final achievementService = Provider.of<AchievementService>(context);

// به‌روزرسانی پیشرفت یک دستاورد
await achievementService.updateProgress('first_workout', 1);

// دریافت دستاوردها
final achievements = achievementService.achievements;

// دریافت دستاوردهای باز شده
final unlocked = achievementService.unlockedAchievements;

// سینک به دیتابیس (معمولاً خودکار است)
await achievementService.syncToDatabase();
```

### 3. سینک خودکار

سرویس به صورت خودکار:
- هنگام بارگذاری صفحه، پیشرفت کاربر را از دیتابیس می‌خواند
- هنگام به‌روزرسانی پیشرفت، در دیتابیس ذخیره می‌کند
- در حالت آفلاین، از cache محلی استفاده می‌کند
- هنگام اتصال مجدد، cache را به دیتابیس سینک می‌کند

## 🔄 سینک و Cache

### Cache محلی

داده‌ها در SharedPreferences با کلید `achievements_cache` ذخیره می‌شوند:
- فرمت: JSON Map از `achievement_id` به `AchievementProgress`
- به‌روزرسانی: خودکار هنگام تغییرات
- پاک‌سازی: هنگام sync موفق به دیتابیس

### سینک خودکار

سرویس به صورت خودکار:
1. هنگام بارگذاری صفحه، از دیتابیس می‌خواند
2. در صورت آفلاین بودن، از cache استفاده می‌کند
3. هنگام تغییرات، هم در cache و هم در دیتابیس ذخیره می‌کند
4. هنگام اتصال مجدد، cache را سینک می‌کند

## 🐛 عیب‌یابی

### مشکل: دستاوردها ذخیره نمی‌شوند

1. بررسی کنید که کاربر لاگین است
2. بررسی کنید که جدول در دیتابیس ایجاد شده است
3. بررسی کنید که RLS policies درست تنظیم شده‌اند
4. لاگ‌های کنسول را بررسی کنید

### مشکل: Cache کار نمی‌کند

1. بررسی کنید که SharedPreferences در دسترس است
2. بررسی کنید که کلید cache درست است
3. لاگ‌های کنسول را بررسی کنید

### مشکل: سینک کار نمی‌کند

1. بررسی کنید که اینترنت متصل است
2. بررسی کنید که کاربر لاگین است
3. بررسی کنید که دیتابیس در دسترس است
4. لاگ‌های کنسول را بررسی کنید

## 📝 نکات مهم

1. **همیشه از `updateProgress` استفاده کنید** - این متد هم در حافظه و هم در دیتابیس به‌روزرسانی می‌کند
2. **سینک خودکار است** - نیازی به فراخوانی دستی `syncToDatabase` نیست مگر در موارد خاص
3. **Cache محلی همیشه در دسترس است** - حتی در حالت آفلاین
4. **RLS Policies مهم هستند** - مطمئن شوید که درست تنظیم شده‌اند

## 🔐 امنیت

- تمام عملیات دیتابیس از طریق RLS محافظت می‌شوند
- کاربران فقط می‌توانند دستاوردهای خود را مشاهده/ویرایش کنند
- Cache محلی فقط برای کاربر فعلی است
- هیچ داده حساسی در cache ذخیره نمی‌شود

## 📊 عملکرد

- **بارگذاری اولیه**: ~100-200ms (بسته به تعداد دستاوردها)
- **به‌روزرسانی**: ~50-100ms (ذخیره در cache + دیتابیس)
- **سینک**: ~200-500ms (بسته به تعداد تغییرات)
