## 🔍 وضعیت فعلی سیستم

### ✅ سیستم‌های موجود

#### 1. سیستم دستاورد (Achievement System)
- ✅ **وضعیت**: پیاده‌سازی شده و فعال
- ✅ **قابلیت‌ها**:
  - دستاوردهای مختلف (تمرین، تغذیه، پیشرفت، اجتماعی، پلتفرم)
  - سیستم امتیازدهی برای هر دستاورد
  - Tier‌های مختلف (برنز، نقره، طلا، پلاتینیوم، الماس)
  - ذخیره‌سازی در دیتابیس Supabase
  - Cache محلی برای عملکرد آفلاین

#### 2. سیستم امتیاز (Score Service)
- ✅ **وضعیت**: پیاده‌سازی شده و فعال
- ✅ **قابلیت‌ها**:
  - ذخیره تاریخچه امتیازات
  - محاسبه امتیاز کل کاربر
  - دسته‌بندی بر اساس منبع (achievement, workout, nutrition, etc.)
  - جدول `point_history` در دیتابیس

#### 3. سیستم Streak (روزهای متوالی)
- ✅ **وضعیت**: پیاده‌سازی شده و فعال
- ✅ **قابلیت‌ها**:
  - ردیابی روزهای متوالی ورود
  - محاسبه longest streak
  - اتصال به دستاوردهای streak
  - ذخیره در جدول `profiles` (login_streak, longest_streak)

#### 4. سیستم رتبه‌بندی مربیان (Trainer Ranking)
- ✅ **وضعیت**: پیاده‌سازی شده و فعال
- ✅ **قابلیت‌ها**:
  - رتبه‌بندی مربیان بر اساس امتیاز کلی
  - الگوریتم محاسبه امتیاز پیشرفته
  - نمایش در صفحه جداگانه
  - **محدودیت**: فقط برای مربیان (role='trainer')

### ❌ سیستم‌های موجود نیست

1. **سیستم رتبه‌بندی کاربران عادی (Athletes)**
   - هیچ سیستم رتبه‌بندی برای کاربران عادی وجود ندارد
   - فقط مربیان رتبه‌بندی می‌شوند

2. **Leaderboard عمومی**
   - هیچ صفحه‌ای برای نمایش رتبه‌بندی کلی کاربران وجود ندارد
   - کاربران نمی‌توانند جایگاه خود را در مقایسه با دیگران ببینند

3. **سیستم لیگ‌بندی**
   - هیچ سیستم لیگ (برنز، نقره، طلا و...) برای کاربران وجود ندارد
   - کاربران در هیچ لیگ خاصی قرار نمی‌گیرند

4. **سیستم رقابت**
   - هیچ مکانیزم رقابتی بین کاربران وجود ندارد
   - کاربران نمی‌توانند با دوستان خود رقابت کنند

## 🎯 تحلیل نیازمندی‌ها

### نیازمندی‌های اصلی

#### 1. رتبه‌بندی کاربران
- کاربران باید بتوانند جایگاه خود را در مقایسه با دیگران ببینند
- رتبه‌بندی باید بر اساس معیارهای عادلانه باشد
- باید به صورت real-time به‌روزرسانی شود

#### 2. سیستم لیگ‌بندی
- کاربران باید در لیگ‌های مختلف قرار بگیرند (برنز، نقره، طلا، پلاتینیوم، الماس)
- ارتقا و سقوط بین لیگ‌ها باید واضح باشد
- هر لیگ باید مزایای خاص خود را داشته باشد

#### 3. Leaderboard
- صفحه نمایش رتبه‌بندی کلی
- فیلتر بر اساس دوستان، لیگ، منطقه و...
- نمایش رتبه کاربر در هر دسته‌بندی

#### 4. رقابت و انگیزه
- ایجاد انگیزه برای فعالیت بیشتر
- رقابت سالم بین کاربران
- نمایش پیشرفت و دستاوردها

### ⚠️ چالش‌های فنی

1. **عملکرد (Performance)**
   - محاسبه رتبه‌بندی برای تعداد زیاد کاربران

2. **مقیاس‌پذیری (Scalability)**
   - با افزایش کاربران، محاسبه رتبه‌بندی کند می‌شود
   - **راه‌حل**: استفاده از Materialized Views
   - **راه‌حل**: محاسبه فقط برای کاربران فعال

3. **همگام‌سازی (Synchronization)**
   - همگام‌سازی رتبه‌بندی بین دستگاه‌ها
   - **راه‌حل**: استفاده از Supabase Realtime
   - **راه‌حل**: Cache invalidation strategy

### ✅ امکان‌سنجی تجاری: **عالی**

#### مزایای تجاری
- ✅ افزایش تعامل کاربران (User Engagement)
- ✅ افزایش Retention Rate
- ✅ ایجاد انگیزه برای استفاده بیشتر از اپ
- ✅ تمایز از رقبا
- ✅ افزایش درآمد از طریق فعالیت بیشتر

#### معیارهای موفقیت

2. **ناامیدی کاربران جدید**
   - کاربران جدید ممکن است احساس کنند نمی‌توانند به رتبه‌های بالا برسند
   - **راه‌حل**: لیگ‌بندی بر اساس سطح تجربه
   - **راه‌حل**: نمایش پیشرفت شخصی به جای فقط رتبه کلی

3. **فشار روانی**
   - برخی کاربران ممکن است تحت فشار قرار بگیرند
   - **راه‌حل**: امکان مخفی کردن رتبه‌بندی
   - **راه‌حل**: تاکید بر پیشرفت شخصی

### ساختار دیتابیس

#### جدول جدید: `user_rankings`

```sql
CREATE TABLE IF NOT EXISTS public.user_rankings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  
  -- امتیاز کلی
  total_score INTEGER NOT NULL DEFAULT 0,
  
  -- رتبه کلی
  global_rank INTEGER,
  
  -- رتبه در لیگ
  league_rank INTEGER,
  
  -- لیگ فعلی
  current_league VARCHAR(20) NOT NULL DEFAULT 'bronze',
  -- مقادیر: bronze, silver, gold, platinum, diamond
  
  -- امتیاز لیگ (برای ارتقا/سقوط)
  league_points INTEGER NOT NULL DEFAULT 0,
  
  -- تاریخ ارتقا/سقوط آخرین لیگ
  league_changed_at TIMESTAMP WITH TIME ZONE,
  
  -- رتبه در لیگ قبلی (برای نمایش پیشرفت)
  previous_league VARCHAR(20),
  
  -- تاریخ آخرین به‌روزرسانی رتبه
  rank_updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  
  -- تاریخ ایجاد
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  
  -- ایندکس منحصر به فرد برای هر کاربر
  UNIQUE(user_id)
);

-- ایندکس‌ها
CREATE INDEX IF NOT EXISTS idx_user_rankings_total_score 
  ON public.user_rankings(total_score DESC);
  
CREATE INDEX IF NOT EXISTS idx_user_rankings_league 
  ON public.user_rankings(current_league, league_points DESC);
  
CREATE INDEX IF NOT EXISTS idx_user_rankings_global_rank 
  ON public.user_rankings(global_rank) 
  WHERE global_rank IS NOT NULL;
```

#### جدول جدید: `leaderboard_snapshots` (اختیاری - برای عملکرد بهتر)

```sql
CREATE TABLE IF NOT EXISTS public.leaderboard_snapshots (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  snapshot_type VARCHAR(50) NOT NULL, -- 'global', 'league_bronze', 'league_silver', etc.
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  rank INTEGER NOT NULL,
  score INTEGER NOT NULL,
  league VARCHAR(20),
  snapshot_date DATE NOT NULL DEFAULT CURRENT_DATE,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  
  UNIQUE(snapshot_type, snapshot_date, user_id)
);

CREATE INDEX IF NOT EXISTS idx_leaderboard_snapshots_type_date 
  ON public.leaderboard_snapshots(snapshot_type, snapshot_date DESC, rank);
```

## 📊 معیارهای امتیازدهی

### فرمول کلی امتیاز

```
Total Score = 
  Achievement Points (30%) +
  Activity Points (25%) +
  Consistency Points (20%) +
  Progress Points (15%) +
  Social Points (10%)
```

### جزئیات هر بخش

#### 1. Achievement Points (30% - امتیاز دستاوردها)
- **منبع**: از `point_history` با `source='achievement'`
- **محاسبه**: مجموع تمام امتیازهای دستاوردها
- **محدودیت**: حداکثر 10,000 امتیاز

#### 2. Activity Points (25% - امتیاز فعالیت)
- **تمرینات انجام شده**: 
  - هر تمرین: 10 امتیاز
  - تمرینات متوالی (streak): +5 امتیاز برای هر روز متوالی
- **وعده‌های غذایی ثبت شده**:
  - هر وعده: 5 امتیاز
  - ثبت کامل روز: +10 امتیاز
- **ورود روزانه**: 5 امتیاز
- **محدودیت**: حداکثر 8,000 امتیاز

#### 3. Consistency Points (20% - امتیاز ثبات)
- **Login Streak**: 
  - هر روز متوالی: 2 امتیاز
  - حداکثر: 1,000 امتیاز (500 روز)
- **Longest Streak**: 
  - هر روز: 1 امتیاز
  - حداکثر: 500 امتیاز
- **محدودیت**: حداکثر 5,000 امتیاز

#### 4. Progress Points (15% - امتیاز پیشرفت)
- **کاهش وزن**:
  - هر کیلوگرم: 50 امتیاز
  - حداکثر: 2,000 امتیاز (40 کیلوگرم)
- **رسیدن به اهداف**:
  - هر هدف: 200 امتیاز
- **بهبود BMI**:
  - هر واحد بهبود: 100 امتیاز
- **محدودیت**: حداکثر 3,000 امتیاز

#### 5. Social Points (10% - امتیاز اجتماعی)
- **دعوت دوستان**:
  - هر دعوت موفق: 50 امتیاز
- **چت با مربی**:
  - هر 10 پیام: 20 امتیاز
- **نظرات و تعاملات**:
  - هر نظر: 5 امتیاز
- **محدودیت**: حداکثر 2,000 امتیاز

### مثال محاسبه

```
کاربری با:
- 5,000 امتیاز دستاورد
- 100 تمرین (1,000 امتیاز)
- 50 روز streak (100 امتیاز)
- کاهش 10 کیلوگرم (500 امتیاز)
- 5 دعوت موفق (250 امتیاز)

Total Score = 
  5,000 (Achievement) +
  1,000 (Activity) +
  100 (Consistency) +
  500 (Progress) +
  250 (Social)
= 6,850 امتیاز
```

## 🏆 سیستم لیگ‌بندی

### ساختار لیگ‌ها

| لیگ | نام فارسی | امتیاز مورد نیاز | رنگ | مزایا |
|-----|----------|-----------------|-----|-------|
| 🥉 **Bronze** | برنز | 0 - 1,000 | #CD7F32 | دسترسی پایه |
| 🥈 **Silver** | نقره | 1,001 - 3,000 | #C0C0C0 | آیکون مخصوص، نشان لیگ |
| 🥇 **Gold** | طلا | 3,001 - 7,000 | #FFD700 | پروفایل طلایی، اولویت پشتیبانی |
| 💎 **Platinum** | پلاتینیوم | 7,001 - 15,000 | #E5E4E2 | پروفایل پلاتینیوم، دسترسی ویژه |
| 💠 **Diamond** | الماس | 15,001+ | #B9F2FF | پروفایل الماس، دسترسی VIP |

### قوانین ارتقا و سقوط

#### ارتقا به لیگ بالاتر
- کاربر باید حداقل امتیاز لیگ بعدی را داشته باشد
- ارتقا فوری است (real-time)
- اعلان و جشن برای ارتقا

#### سقوط به لیگ پایین‌تر
- اگر امتیاز کاربر زیر حداقل لیگ فعلی برسد
- **قانون محافظت**: کاربران جدید (کمتر از 30 روز) محافظت می‌شوند
- **قانون grace period**: 7 روز فرصت برای بازگشت

### League Points (امتیاز لیگ)

برای رقابت درون لیگ، از League Points استفاده می‌شود:

```
League Points = Total Score - Minimum Score of Current League
```

مثال:
- کاربر با 5,000 امتیاز در لیگ Gold
- حداقل امتیاز Gold: 3,001
- League Points = 5,000 - 3,001 = 1,999

## 📈 Leaderboard

### انواع Leaderboard

#### 1. Global Leaderboard (رتبه‌بندی کلی)
- نمایش 100 کاربر برتر
- مرتب‌سازی بر اساس Total Score
- به‌روزرسانی: هر ساعت

#### 2. League Leaderboard (رتبه‌بندی لیگ)
- نمایش کاربران در همان لیگ
- مرتب‌سازی بر اساس League Points
- نمایش رتبه کاربر در لیگ

#### 3. Friends Leaderboard (رتبه‌بندی دوستان)
- فقط دوستان کاربر
- مرتب‌سازی بر اساس Total Score
- Real-time updates

#### 4. Weekly/Monthly Leaderboard
- رتبه‌بندی هفتگی/ماهانه
- Reset در ابتدای هر هفته/ماه
- جوایز ویژه برای برندگان

### فیلترها و جستجو

- فیلتر بر اساس لیگ
- جستجوی کاربر خاص
- نمایش رتبه کاربر فعلی
- نمایش کاربران نزدیک (±10 رتبه)

## ✅ مزایا و معایب

### ✅ مزایا

#### 1. افزایش تعامل (Engagement)
- کاربران برای بالا بردن رتبه بیشتر فعالیت می‌کنند
- ایجاد انگیزه برای استفاده مداوم از اپ

#### 2. افزایش Retention
- کاربران برای حفظ رتبه خود برمی‌گردند
- Streak system با رتبه‌بندی ترکیب می‌شود

#### 3. تمایز از رقبا
- ویژگی منحصر به فرد
- ایجاد حس جامعه و رقابت

#### 4. افزایش درآمد
- کاربران فعال‌تر = استفاده بیشتر از ویژگی‌های پولی
- امکان فروش آیتم‌های مخصوص لیگ‌های بالا

#### 5. داده‌های ارزشمند
- تحلیل رفتار کاربران
- شناسایی کاربران برتر
- برنامه‌ریزی بهتر

### ⚠️ معایب و چالش‌ها

#### 1. ریسک تقلب
- کاربران ممکن است برای بالا بردن رتبه تقلب کنند
- **راه‌حل**: سیستم اعتبارسنجی و تشخیص تقلب

#### 2. فشار روانی
- برخی کاربران ممکن است تحت فشار قرار بگیرند
- **راه‌حل**: امکان مخفی کردن رتبه، تاکید بر پیشرفت شخصی

#### 3. ناامیدی کاربران جدید
- کاربران جدید ممکن است احساس کنند نمی‌توانند به رتبه‌های بالا برسند

### الگوریتم محاسبه رتبه

```dart
class RankingService {
  // محاسبه امتیاز کل کاربر
  Future<int> calculateTotalScore(String userId) async {
    // 1. Achievement Points (30%)
    final achievementPoints = await _getAchievementPoints(userId);
    
    // 2. Activity Points (25%)
    final activityPoints = await _getActivityPoints(userId);
    
    // 3. Consistency Points (20%)
    final consistencyPoints = await _getConsistencyPoints(userId);
    
    // 4. Progress Points (15%)
    final progressPoints = await _getProgressPoints(userId);
    
    // 5. Social Points (10%)
    final socialPoints = await _getSocialPoints(userId);
    
    return achievementPoints + activityPoints + 
           consistencyPoints + progressPoints + socialPoints;
  }
  
  // به‌روزرسانی رتبه همه کاربران
  Future<void> updateAllRankings() async {
    // فقط برای کاربران فعال (آخرین فعالیت در 30 روز گذشته)
    final activeUsers = await _getActiveUsers();
    
    for (final user in activeUsers) {
      final score = await calculateTotalScore(user.id);
      await _updateUserRanking(user.id, score);
    }
    
    // به‌روزرسانی رتبه‌های global
    await _updateGlobalRanks();
    
    // به‌روزرسانی رتبه‌های لیگ
    await _updateLeagueRanks();
  }
}
```

#### 2. محاسبه دوره‌ای

- به‌روزرسانی رتبه‌های لیگ: هر 6 ساعت

- استفاده از Materialized Views برای Leaderboard
- Refresh دوره‌ای (هر ساعت)

## ⚠️ نکات مهم و ریسک‌ها

### نکات امنیتی

1. **RLS Policies**
   - کاربران فقط می‌توانند رتبه خود را ببینند
   - Leaderboard عمومی برای همه قابل مشاهده است
   - اطلاعات حساس (مثل امتیاز دقیق) فقط برای کاربر خودش

2. **اعتبارسنجی داده‌ها**
   - بررسی صحت داده‌های ورودی
   - جلوگیری از دستکاری امتیازها
   - Logging تمام تغییرات

3. **Rate Limiting**
   - محدود کردن تعداد درخواست‌های رتبه‌بندی
   - جلوگیری از DDoS

### نکات عملکردی

1. **مقیاس‌پذیری**
   - با افزایش کاربران، نیاز به بهینه‌سازی بیشتر
   - استفاده از Pagination برای Leaderboard
   - Lazy Loading برای لیست‌های طولانی

2. **حالت آفلاین**
   - Cache رتبه کاربر در دستگاه
   - نمایش رتبه از Cache هنگام آفلاین
   - Sync هنگام اتصال مجدد

### نکات UX

1. **شفافیت**
   - نمایش واضح معیارهای امتیازدهی
   - نمایش پیشرفت به سمت لیگ بعدی
   - توضیح قوانین ارتقا/سقوط

2. **انگیزه**
   - اعلان‌های ارتقا لیگ
   - نمایش دستاوردها
   - تاکید بر پیشرفت شخصی

3. **گزینه‌های کاربر**
   - امکان مخفی کردن رتبه
   - امکان غیرفعال کردن اعلان‌ها
   - تنظیمات حریم خصوصی
